Overview

PySpark is the Python API for Apache Spark, an open-source, distributed computing system that provides an interface for programming entire clusters with implicit data parallelism and fault tolerance.

Originally developed at UC Berkeley's AMPLab in 2009 and later donated to Apache, Spark processes data at massive scale — up to 100× faster than Hadoop MapReduce through in-memory computing.

Architecture

Spark follows a master-worker architecture. The driver program runs the main function and coordinates distributed operations across worker nodes via a cluster manager.

Key Components

Driver Program
Runs the main() function, creates SparkContext, converts user code into tasks, and schedules them on executors.
SparkContext
Entry point for Spark functionality. Connects to cluster manager, creates RDDs, accumulators, and broadcast variables.
Executor
JVM process on worker nodes. Runs tasks assigned by driver, reports results, stores data in cache.
DAG Scheduler
Converts logical execution plan into a directed acyclic graph of stages, optimising for data locality.
Task Scheduler
Assigns individual tasks from stages to executor slots, respecting data locality preferences.
Cluster Manager
Allocates resources. Options: Standalone, Apache YARN, Apache Mesos, or Kubernetes.

DAG Execution Model

Spark builds a Directed Acyclic Graph (DAG) of operations before executing. This allows query optimization, fault recovery, and stage-level parallelism.

Stage 1 — Read + Map
Read data from HDFS/S3 → apply map transformations (narrow dependency)
Shuffle Boundary
Wide dependency (groupBy, join) — data redistribution across network
Stage 2 — Aggregate
Aggregation, reduce, sort on shuffled partitions
Action — Write Output
Trigger execution; write results to sink (file, database, stream)

Resilient Distributed Datasets (RDDs)

RDDs are the fundamental data structure in Spark — an immutable, partitioned collection of elements that can be operated on in parallel. RDDs provide fault tolerance through lineage (recomputing lost data from its transformation history).

⚠️ Note In most cases, prefer DataFrames over RDDs. DataFrames leverage the Catalyst query optimizer and Tungsten execution engine for much better performance. Use RDDs only when you need fine-grained control.

RDD Properties

Immutable
Once created, RDDs cannot be changed. Transformations create new RDDs.
Partitioned
Data is split across partitions distributed on worker nodes.
Lazy
Transformations are deferred; computed only when an action is called.
Fault-Tolerant
Lost partitions can be recomputed using the lineage graph.

Partitioning

Each RDD is divided into partitions. Each partition is processed by one task on one executor core.

Creating RDDs

# From Python collection
rdd1 = sc.parallelize([1, 2, 3, 4, 5], numSlices=4)

# From text file (each line = one element)
rdd2 = sc.textFile("hdfs://path/to/file.txt")
rdd3 = sc.textFile("s3a://bucket/data/*.csv")

# From multiple files
rdd4 = sc.wholeTextFiles("/data/logs/")  # (filename, content)

# Check number of partitions
print(rdd1.getNumPartitions())  # 4

RDD Transformations

numbers = sc.parallelize(range(1, 11))

# map — apply function to each element
squared = numbers.map(lambda x: x ** 2)

# filter — keep elements matching predicate
evens = numbers.filter(lambda x: x % 2 == 0)

# flatMap — map then flatten
words = sc.parallelize(["hello world", "foo bar"])
tokens = words.flatMap(lambda line: line.split())

# reduceByKey — aggregate by key
pairs = sc.parallelize([("a", 1), ("b", 2), ("a", 3)])
sums  = pairs.reduceByKey(lambda x, y: x + y)  # [("a",4),("b",2)]

# groupByKey — group values by key
groups = pairs.groupByKey()

# sortByKey
sorted_pairs = pairs.sortByKey(ascending=True)

# distinct
unique = numbers.distinct()

# union, intersection, subtract
rdd_a = sc.parallelize([1, 2, 3])
rdd_b = sc.parallelize([2, 3, 4])
union_rdd = rdd_a.union(rdd_b)
inter_rdd = rdd_a.intersection(rdd_b)
diff_rdd  = rdd_a.subtract(rdd_b)

RDD Actions

rdd = sc.parallelize([3, 1, 4, 1, 5, 9])

# collect — return all elements to driver
result = rdd.collect()            # [3,1,4,1,5,9]

# count — number of elements
n = rdd.count()                   # 6

# first, take
first = rdd.first()              # 3
top3  = rdd.take(3)             # [3,1,4]
top3s = rdd.top(3)              # [9,5,4]

# reduce
total = rdd.reduce(lambda a, b: a + b)   # 23

# aggregate — more flexible reduce
stats = rdd.aggregate(
    (0, 0),
    lambda acc, x: (acc[0] + x, acc[1] + 1),
    lambda a, b: (a[0] + b[0], a[1] + b[1])
)

# countByValue, countByKey
vc = rdd.countByValue()   # {3:1, 1:2, 4:1, 5:1, 9:1}

# saveAsTextFile
rdd.saveAsTextFile("hdfs://output/path")

DataFrames

A DataFrame is a distributed collection of data organized into named columns — conceptually equivalent to a table in a relational database or a Pandas DataFrame, but distributed. DataFrames use Spark's Catalyst optimizer and Tungsten engine for dramatic performance improvements over RDDs.

Creating DataFrames

from pyspark.sql import SparkSession
from pyspark.sql.types import *
from pyspark.sql import functions as F

# ── From Python list ─────────────────────────────
data = [("Alice", 34, "Engineering"),
        ("Bob",   28, "Marketing"  ),
        ("Carol", 41, "Engineering")]

schema = StructType([
    StructField("name",       StringType(),  nullable=False),
    StructField("age",        IntegerType(), nullable=True),
    StructField("department", StringType(),  nullable=True),
])

df = spark.createDataFrame(data, schema=schema)

# ── From CSV ──────────────────────────────────────
df_csv = spark.read \
    .option("header",          "true") \
    .option("inferSchema",     "true") \
    .option("nullValue",       "NA")   \
    .csv("data/employees.csv")

# ── From JSON / Parquet / ORC ─────────────────────
df_json    = spark.read.json("data/events.json")
df_parquet = spark.read.parquet("data/sales/")
df_orc     = spark.read.orc("data/logs/")

# ── From Pandas ───────────────────────────────────
import pandas as pd
pdf = pd.DataFrame({"x": [1,2,3], "y": [4,5,6]})
df_from_pandas = spark.createDataFrame(pdf)

DataFrame Operations

# ── Inspection ────────────────────────────────────
df.show(5)              # print first 5 rows
df.printSchema()        # print schema tree
df.dtypes               # list of (name, type) tuples
df.describe().show()    # descriptive statistics
df.count()              # row count (action)
df.columns             # list of column names

# ── Selection ─────────────────────────────────────
df.select("name", "age")
df.select(df.name, df.age + 1)
df.select(F.col("age") * 2)

# ── Filtering ─────────────────────────────────────
df.filter(F.col("age") > 30)
df.filter((df.age > 25) & (df.department == "Engineering"))
df.where("age > 30")    # SQL-string syntax

# ── Adding / Renaming columns ──────────────────────
df.withColumn("age_squared", F.col("age") ** 2)
df.withColumnRenamed("name", "employee_name")
df.drop("department")

# ── Sorting ───────────────────────────────────────
df.orderBy(F.col("age").desc())
df.sort("department", "age")

# ── Grouping & Aggregation ────────────────────────
df.groupBy("department") \
  .agg(F.count("*").alias("headcount"),
       F.avg("age").alias("avg_age"),
       F.max("age").alias("max_age")) \
  .show()

# ── Joins ─────────────────────────────────────────
dept_df = spark.createDataFrame(
    [("Engineering", "San Francisco"),
     ("Marketing",   "New York")],
    ["dept", "location"])

joined = df.join(dept_df, df.department == dept_df.dept, "left")

# ── Writing ───────────────────────────────────────
df.write \
  .mode("overwrite") \
  .partitionBy("department") \
  .parquet("output/employees/")

Spark SQL

Spark SQL lets you query structured data using standard SQL, mixing SQL expressions with DataFrame/Dataset operations. It uses the same execution engine, so performance is identical regardless of API used.

# Register as temp view
df.createOrReplaceTempView("employees")

# Run SQL queries
result = spark.sql("""
    SELECT
        department,
        COUNT(*)          AS headcount,
        AVG(salary)       AS avg_salary,
        MAX(salary)       AS max_salary,
        PERCENTILE(salary, 0.5) AS median_salary
    FROM employees
    WHERE hire_date >= '2020-01-01'
    GROUP BY department
    HAVING COUNT(*) > 5
    ORDER BY avg_salary DESC
""")

result.show()

# Global temp views (persist across sessions)
df.createOrReplaceGlobalTempView("global_employees")
spark.sql("SELECT * FROM global_temp.global_employees")

# Caching for repeated queries
spark.catalog.cacheTable("employees")
df.cache()          # lazy cache
df.persist()        # alias for cache()

# Show execution plan
result.explain(extended=True)    # parsed → analyzed → optimized → physical

Catalyst Query Optimizer

UDFs — User Defined Functions

from pyspark.sql.functions import udf
from pyspark.sql.types import StringType

# Register Python UDF (slower — crosses JVM boundary)
def clean_phone(phone):
    import re
    return re.sub(r"[^\d]", "", phone) if phone else None

clean_phone_udf = udf(clean_phone, StringType())
df.withColumn("clean_phone", clean_phone_udf("phone_raw"))

# Pandas UDF (vectorized — much faster)
from pyspark.sql.functions import pandas_udf
import pandas as pd

@pandas_udf("double")
def log_transform(series: pd.Series) -> pd.Series:
    import numpy as np
    return np.log1p(series)

df.withColumn("log_revenue", log_transform("revenue"))

# Register UDF for SQL use
spark.udf.register("clean_phone", clean_phone, StringType())
spark.sql("SELECT clean_phone(phone) FROM users")

Structured Streaming

Structured Streaming treats a live data stream as a continuously appended table. You write the same DataFrame batch queries and Spark handles the incremental execution, state management, and exactly-once guarantees automatically.

# ── Read from Kafka ────────────────────────────────
stream_df = spark.readStream \
    .format("kafka") \
    .option("kafka.bootstrap.servers", "broker:9092") \
    .option("subscribe", "events") \
    .option("startingOffsets", "latest") \
    .load()

# Deserialize JSON payload
from pyspark.sql.types import StructType, StringType, LongType
from pyspark.sql.functions import from_json, col

schema = StructType() \
    .add("user_id", LongType()) \
    .add("event",   StringType()) \
    .add("ts",      LongType())

events = stream_df \
    .selectExpr("CAST(value AS STRING) as json_str") \
    .select(from_json(col("json_str"), schema).alias("data")) \
    .select("data.*")

# ── Aggregation with watermark ─────────────────────
from pyspark.sql.functions import window, count

windowed = events \
    .withWatermark("event_time", "10 minutes") \
    .groupBy(
        window(col("event_time"), "5 minutes"),
        col("event")
    ) \
    .agg(count("*").alias("event_count"))

# ── Write stream ───────────────────────────────────
query = windowed.writeStream \
    .outputMode("update") \
    .format("parquet") \
    .option("path",       "s3://bucket/output/") \
    .option("checkpointLocation", "s3://bucket/checkpoint/") \
    .trigger(processingTime="30 seconds") \
    .start()

query.awaitTermination()

from pyspark.ml import Pipeline
from pyspark.ml.feature import (
    StringIndexer, OneHotEncoder,
    VectorAssembler, StandardScaler
)
from pyspark.ml.classification import RandomForestClassifier
from pyspark.ml.evaluation import BinaryClassificationEvaluator
from pyspark.ml.tuning import CrossValidator, ParamGridBuilder

# ── Feature engineering ────────────────────────────
indexer = StringIndexer(
    inputCols=["category", "country"],
    outputCols=["category_idx", "country_idx"]
)

encoder = OneHotEncoder(
    inputCols=["category_idx", "country_idx"],
    outputCols=["category_ohe", "country_ohe"]
)

assembler = VectorAssembler(
    inputCols=["age", "income", "category_ohe", "country_ohe"],
    outputCol="raw_features"
)

scaler = StandardScaler(
    inputCol="raw_features",
    outputCol="features"
)

# ── Classifier ────────────────────────────────────
rf = RandomForestClassifier(
    labelCol="label",
    featuresCol="features",
    numTrees=100,
    maxDepth=10
)

# ── Build pipeline ────────────────────────────────
pipeline = Pipeline(stages=[indexer, encoder, assembler, scaler, rf])

# ── Hyperparameter tuning ─────────────────────────
param_grid = ParamGridBuilder() \
    .addGrid(rf.numTrees,  [50, 100, 200]) \
    .addGrid(rf.maxDepth,  [5,  10]) \
    .build()

evaluator = BinaryClassificationEvaluator(metricName="areaUnderROC")

cv = CrossValidator(
    estimator=pipeline,
    estimatorParamMaps=param_grid,
    evaluator=evaluator,
    numFolds=5,
    parallelism=4    # train folds in parallel
)

# ── Train & evaluate ──────────────────────────────
train, test = df.randomSplit([0.8, 0.2], seed=42)
model = cv.fit(train)
predictions = model.transform(test)

auc = evaluator.evaluate(predictions)
print(f"AUC: {auc:.4f}")

# ── Save & load model ─────────────────────────────
model.bestModel.save("s3://bucket/models/churn_model_v1")
from pyspark.ml.pipeline import PipelineModel
loaded = PipelineModel.load("s3://bucket/models/churn_model_v1")

Key Performance Levers

1. Partitioning Strategy

# Check current partitions
print(df.rdd.getNumPartitions())

# Repartition (full shuffle)
df_repartitioned = df.repartition(200)
df_by_col = df.repartition(50, "user_id")  # hash partition by col

# Coalesce (reduce partitions, no shuffle)
df_coalesced = df.coalesce(10)  # only decreases partitions

# Rule of thumb: 128 MB per partition
# Total partitions = ceil(data_size_bytes / 128MB)

2. Caching & Persistence

from pyspark import StorageLevel

# Cache (memory only, default)
df.cache()
df.persist()

# Custom storage levels
df.persist(StorageLevel.MEMORY_AND_DISK)    # spill to disk if needed
df.persist(StorageLevel.MEMORY_ONLY_SER)     # serialized (less RAM)
df.persist(StorageLevel.DISK_ONLY)           # disk only
df.persist(StorageLevel.MEMORY_AND_DISK_2)   # replicated ×2

# Unpersist when done
df.unpersist()

3. Broadcast Joins

from pyspark.sql.functions import broadcast

# For small lookup tables (< 200 MB)
result = large_df.join(
    broadcast(small_lookup_df),
    "join_key"
)

# Auto-broadcast threshold (default: 10 MB)
spark.conf.set("spark.sql.autoBroadcastJoinThreshold", 50 * 1024 * 1024)  # 50 MB

4. Skew Handling

# Enable adaptive query execution (Spark 3.0+)
spark.conf.set("spark.sql.adaptive.enabled", "true")
spark.conf.set("spark.sql.adaptive.skewJoin.enabled", "true")
spark.conf.set("spark.sql.adaptive.coalescePartitions.enabled", "true")

# Manual skew mitigation: salt the key
import pyspark.sql.functions as F
import random

SALT = 50
df_salted = large_df.withColumn(
    "salted_key",
    F.concat(F.col("skewed_key"), F.lit("_"), (F.rand() * SALT).cast("int"))
)

Best Practices

Do ✅

# ✅ Use DataFrames over RDDs for structured data
df.filter(F.col("age") > 30)  # pushes predicate to storage

# ✅ Filter early — reduce data as soon as possible
spark.read.parquet("huge_table/") \
    .filter("date = '2024-01-01'") \  # partition pruning
    .select("id", "value")           # projection pushdown

# ✅ Use Parquet/ORC with compression
df.write.mode("overwrite").parquet("path/")

# ✅ Avoid collect() on large DataFrames
df.limit(1000).toPandas()  # safer

# ✅ Use broadcast for small lookup tables
result = big_df.join(broadcast(small_df), "key")

# ✅ Specify schema explicitly (avoid inferSchema)
df = spark.read.schema(my_schema).csv("data.csv")

# ✅ Use Pandas UDFs instead of Python UDFs
@pandas_udf("double")
def vectorized_fn(s: pd.Series) -> pd.Series: ...

# ✅ Enable AQE for automatic optimizations
spark.conf.set("spark.sql.adaptive.enabled", "true")

Don't ❌

# ❌ Avoid collect() on large data — OOM risk
all_data = df.collect()  # DANGEROUS on 100 GB tables

# ❌ Don't use Python UDFs when built-in functions exist
# Bad:
udf_upper = udf(lambda x: x.upper(), StringType())
# Good:
F.upper("name")

# ❌ Don't repartition unnecessarily (costly shuffle)
df.repartition(200).repartition(50)  # two shuffles!

# ❌ Don't create too many small files
df.write.parquet("path/")   # may write 200 tiny files
# Better:
df.coalesce(10).write.parquet("path/")

# ❌ Don't use groupByKey on RDDs — prefer reduceByKey
# Bad (shuffles all data):
rdd.groupByKey().map(lambda k, v: (k, sum(v)))
# Good (pre-aggregates per partition):
rdd.reduceByKey(lambda a, b: a + b)
